### Steps to Set Up a Proxy for Vault

1. **Set up an EC2 instance:**
   Use the [quick-ec2-tf](https://github.com/tallen-hashicorp/quick-ec2-tf) repository to provision an EC2 instance.

2. **Configure DNS:**
   Set up an A record on your personal Route 53 domain pointing to the EC2 instance, e.g., `vault.the-tech-tutorial.com`.

3. **Install Nginx and Certbot:**
   Update the system and install Nginx and Certbot for SSL management:
   ```bash
   sudo apt update
   sudo apt install nginx certbot python3-certbot-nginx
   ```

4. **Prepare Nginx Configuration:**
   Copy the `default-pre` Nginx config file to the appropriate directory:
   ```bash
   sudo cp ./jumpbox/default-pre /etc/nginx/sites-enabled/default
   ```

5. **Obtain an SSL Certificate:**
   Use Certbot to obtain a Let’s Encrypt SSL certificate for your domain:
   ```bash
   sudo certbot --nginx -d vault.the-tech-tutorial.com
   ```

6. **Update Nginx Configuration:**
   Modify the Nginx config file to reflect the correct domain and certificate locations:
   - Replace `vault.the-tech-tutorial.com` with the public DNS of the EC2 instance (lines 3 and 25 in the `default` configuration).
   - Update the paths to the SSL certificate files generated by Certbot.

7. **Deploy the Updated Nginx Configuration:**
   Copy the updated `default` configuration file to the server:
   ```bash
   sudo cp ./jumpbox/default /etc/nginx/sites-enabled/default
   ```

8. **Open an SSH Tunnel:**
   Run the following command to create a reverse SSH tunnel, forwarding traffic to the Vault instance:
   ```bash
   ssh -R 0.0.0.0:8200:127.0.0.1:8200 ubuntu@34.198.16.120
   ```

9. **Access Vault:**
   Access Vault using the following URL:
   ```
   https://vault.the-tech-tutorial.com:8220/
   ```

> **Note:** This setup is not recommended for production environments. It’s best suited for local or test use cases only.
